We just shipped MCP token linting tools for zeroheight — and they change how you keep AI-generated code aligned with your design system.

Instead of manually hunting for hard-coded values like `#F03E3E` or `16px`, you can now lint your code directly against your token sets in zeroheight and get suggested replacements instantly — or auto-apply them.

🔍 What it does:
• Scans code for hard-coded colors and spacing values
• Compares them against your linked zeroheight token sets
• Flags violations with the closest matching token
• Auto-applies fixes before you merge

⚙️ How to set it up:
1. Go to Styleguide Settings → MCP and Markdown
2. Link one or more token sets
3. In your MCP client (e.g. Cursor), ask it to "lint this code using zeroheight"

This helps to close the loop on AI-generated code: generate → lint → fix → ship.
