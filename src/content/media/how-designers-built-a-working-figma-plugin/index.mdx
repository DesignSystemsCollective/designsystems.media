Can designers really build a working Figma plugin in 48 hours? Christoph Hellmuth (NordVPN), Raquel Pereira (Nokia) and Casandra Sandu (Trackunit) walk through exactly how their team of 10 designers built Swap Wizard during the Into Design Systems AI Hackathon, won first place out of 150 designers and shipped a real plugin that solves component library migrations in Figma.

You'll see the full process: how they scoped the problem, planned before prompting, used Cursor's Plan, Agent, Ask and Debug modes, set up GitHub for async collaboration across time zones, built a landing page with Lovable, and refactored 4,000 lines of AI-generated spaghetti into a production-ready codebase. They also share the free Figma Plugin Developer Skill that saves you tokens and prevents the most common plugin bugs from day one.

This session was presented at the AI Conference for Designers 2026 by Into Design Systems.

💡 Key Points

• Every minute you spend planning before prompting saves an hour of debugging later. The team spent the first few hours of a 48-hour hackathon not writing any code.

• A vague prompt cost them $9 over two minutes with low precision. A clear, structured prompt with the right model did the same task in seconds for almost nothing. 500x cheaper.

• Use the right Cursor mode for the job: Plan mode to scope, Agent mode to build, Ask mode to learn, Debug mode to fix root causes instead of guessing.

• Designers are already great at spotting problems. Identifying the bug is 80% of the work. AI just needs the right context, screenshots and logs to fix it.

• AI Skills work like Legos. They're modular, plug-and-play and turn your generalist agent into a domain expert. But always read a skill before installing it. Prompt injection is real.

• You don't need engineering skills to ship. You need good scope, the right tools and the discipline to commit often and roll back when things break.
